The source roadmap lives under `reports/`; this tracked doc keeps release evidence reproducible from a clean checkout without committing generated report folders.

## Scope

- Source requirement: `reports/docs/v2.3.0/tron-ide-roadmap.md`, section `## 3. 短期路线图：v2.3.0 发布可信度与 TRON 基础闭环`.

- Scope end: immediately before `## 4. 中期路线图：补齐 TRON 开发者完整工作流`.
- Release principle: local automated coverage is evidence for local code paths; external P0 evidence such as Nile deployment, real TronLink authorization, account switching, and network switching must be recorded as external/manual evidence and remains blocked if the environment cannot provide the wallet/network.

## Explicit blocked / external evidence

- Real TronLink authorization, account switching, network switching, and Nile deploy/call smoke cannot be completed without a browser profile that has TronLink installed, unlocked, and funded on Nile.
- Nightwatch landing E2E cannot run unless Selenium/WebDriver is available on `localhost:4444` or the Nightwatch service is configured to start it.
- Mainnet is not a pass criterion for this release evidence; Nile/testnet is the intended runtime validation target.
